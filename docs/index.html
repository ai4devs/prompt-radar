About the tool

Developer-facing feedback for prompts-as-code.

Prompts increasingly shape the behavior of LLM-enabled software. Yet they are often maintained as plain strings with little support from IDEs, linters, or static analysis tools. Prompt Radar brings prompt-quality inspection directly into VS Code.

🧭

Developer-in-the-loop

Prompt Radar does not scan a project automatically. The developer selects one or more prompt fragments and runs the analysis when needed.

📡

Radar-based explanation

The tool summarizes the analysis using a radar chart, helping developers quickly see which smell dimensions deserve attention.

🛠️

Actionable IDE feedback

Results are displayed inside VS Code so developers can review possible issues and revise prompts without leaving their workflow.

Smell categories

Five dimensions of prompt quality.

Prompt Radar flags possible issues as inspection signals, not definitive defects. Each signal helps developers decide whether a prompt needs revision.

SecuritySecrets, leakage, prompt injection, or unsafe access.
SafetyPotentially harmful, unethical, biased, or policy-risky outputs.
ReliabilityAmbiguous, underspecified, or unstable instructions.
CostRedundant context, verbosity, or unnecessary token usage.
FormatWeak output constraints or missing parseable structure.

How it works

A simple workflow inside VS Code.

Select

The developer highlights a prompt, prompt template, or chat message in the editor.

Analyze

Prompt Radar sends the selected text to the LLM-based prompt smell analyzer.

Visualize

The extension renders the smell profile as a radar chart across the five categories.

Revise

The developer reviews the explanation, updates the prompt, and can re-run the analysis.
